## 对比实验：没有框架 vs 有框架

### 场景

用户问了一个需要"持续上下文"的问题：

> **"我正在写一个科幻小说，主角是科学家，在研究量子计算时遇到了伦理困境，怎么设计接下来的冲突比较好？"**

---

## ❌ 没有 NeuroCortex（普通 AI 对话）

```
用户：我正在写一个科幻小说，主角是科学家，在研究量子计算时遇到了伦理困境，怎么设计接下来的冲突比较好？

AI：这个问题很有意思！以下是一些建议：
1. 可以让主角遇到一个更强大的竞争对手...
2. 或者让他陷入内心的道德挣扎...
3. 也可以加入一个神秘组织...

（用户需要重新解释：这是科幻世界观、科学设定、主角背景等）
```

**问题**：
- ❌ AI 不知道你的世界观
- ❌ 每次都要重新解释背景
- ❌ 回答泛泛而谈，没有针对性

---

## ✅ 有 NeuroCortex（加载框架后）

```
用户：我正在写一个科幻小说，主角是科学家，在研究量子计算时遇到了伦理困境，怎么设计接下来的冲突比较好？

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🧠 NeuroCortex 已激活
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[加载域] writing (priority=7, context_boost=3)
[触发直觉] INT-003: 敏感内容 → 确保100%虚拟化
[加载知识] domains/creative-writing.md
[加载知识] domains/science-fiction.md

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

AI：好问题。结合科幻小说的创作规范，我建议从以下几个角度设计冲突：

**1. 内部冲突（推荐）**
科学家的伦理困境通常源于"知识与责任的冲突"——
他想推动科学进步，但发现成果可能被滥用。
这与科幻小说中常见的"弗兰肯斯坦困境"形成呼应。

**2. 外部冲突**
量子计算如果成功，可能会打破现有的加密体系，
导致社会动荡。主角面临选择：公开发布还是保密？

**3. 时间压力**
竞争对手也在研究同类课题，如果对手先成功，
主角的研究将失去价值，甚至可能成为"威胁"。

你倾向于哪个方向？或者我可以详细展开其中一个？
```

**效果**：
- ✅ AI 知道你的创作域（creative-writing）
- ✅ 自动加载相关知识
- ✅ 回答有针对性，与既有创作规范一致
- ✅ 主动触发直觉规则（确保虚拟化）

## 第二个场景：域冲突裁决演示

### 用户问

> "这个 SQL 查询太慢了，怎么优化？"

---

## ❌ 没有域冲突裁决（普通 AI）

```
AI：这个SQL慢的原因可能是：
1. 没有建索引
2. 数据量太大
3. 查询语句写得不优

建议加索引、限制返回条数、优化JOIN...
```

**问题**：
- 没有利用你项目已有的数据库规范
- 可能给出不适用于你项目的建议

---

## ✅ 有域冲突裁决（智能加载）

```
用户：这个 SQL 查询太慢，怎么优化？

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🧠 NeuroCortex 启动序列
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[Step 1] 安全检查 ✅
[Step 2] 注意力扫描 → 匹配域：coding(8) + data(7)
         ↓
         域冲突检测！执行三级裁决：
         ① priority：coding(8) > data(7)
         ② 上下文窗口：无上一轮，skip
         ③ 裁决结果：加载 coding 域

[Step 3] 跨域引用 → @ref:data/performance
[Step 4] INTUITION 扫描 → INT-002 触发（高危操作）⚠️
         → 先扫描再报告，风险提示

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

AI：在优化之前，先确认几点：

⚠️ **高危操作提醒**：修改查询前建议备份

**根据 coding 域的项目规范**（@ref:data/performance）：

1. **先看执行计划**（你用的是 PostgreSQL？）
   EXPLAIN ANALYZE SELECT ...;

2. **根据计划判断**：
   - 如果 `Seq Scan` 多 → 考虑加索引
   - 如果 `Hash Join` 慢 → 检查统计信息

3. **通用规范**：
   - 超过100ms的查询需要记录
   - 禁止全表扫描除非数据量<1000行

需要我帮你分析具体的执行计划吗？
```

**效果**：
- ✅ 自动识别高危操作，触发风险提示
- ✅ 加载了通用的数据库优化规范
- ✅ 给出针对性的优化建议

## 第三个场景：会话结束，AI 自动维护

### 用户说

> "谢谢，就到这里吧"

---

```

```

**问题**：
- 没有记录这次对话
- 下次要从零开始

- ✅ AI 自动更新了 INTUITION 规则
