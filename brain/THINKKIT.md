## 工具目录

**🔍 分析型**
- 第一性原理：追问本质，不被表象迷惑
- 费曼检验：能否向外行解释清楚
- 逆向思维：从目标反推路径
- 还原论：拆解到最小可验证单元

**🎨 创造型**
- 跨学科碰撞：不同领域知识随机组合
- 类比迁移：A域解法→B域问题
- 极端假设：如果…会怎样？
- 否定之否定：先否定再重建

**🛡️ 验证型**
- 魔鬼代言人：主动寻找反例
- 边界测试：极端条件下是否成立
- 一致性检查：是否与已知事实矛盾
- 奥卡姆剃刀：是否有更简单的解释

**🧭 导向型**
- 目标锚定：当前行动是否服务核心目标
- 机会成本：选择A意味着放弃了什么
- 二阶效应：后续影响是什么
- 正向引导：从成功经验提炼可复用模式

## 使用示例

### 第一性原理

```
问题：为什么需要做这个？
追问：这件事的物理/逻辑本质上由什么决定？
验证：不接受"大家都这么做"作为理由
```

### 费曼检验

```
问题：能向一个完全不懂的人解释清楚吗？
检测：发现"解释不清"的地方 = 还没真正理解
方法：用类比和简化代替术语堆砌
```

### 逆向思维

```
问题：不知道从哪开始？
倒推：如果成功了，那是什么样？
继续：从目标一步步往回推，找到第一个行动
```

## 案例贡献指南

添加新案例：

1. 在 `brain/cases/` 创建 `.md` 文件
2. 命名格式：`方法名_案例N.md`
3. 文件结构：

```markdown
# 案例：（方法名）— （一句话描述）

> **方法**：（方法名）
> **适用场景**：（一句话描述）

## 问题背景
（用户遇到的真实问题）

## 执行步骤
（如何应用这个方法）

## 效果对比
（使用前 vs 使用后）

## 关键洞察
（核心经验总结）
```
