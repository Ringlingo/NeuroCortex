```

        │
        ▼

        ├─ 检查是否有待处理的配置写入请求
        ├─ 如有：执行 META.md 安全规则检查
        └─ 通过 → 继续；拒绝 → 输出不可变声明，终止
        │
        ▼
Step 2: 【注意力扫描】
        ├─ 扫描 FOCUS.md 关键词，匹配所有命中域
        ├─ 如只有一个域命中 → 直接加载
        └─ 如多个域命中 → 执行三级裁决：
             ① priority 权重比较
             ② 上下文窗口延续（窗口期3轮，+3加成）
             ③ 用户显式指令（最高优先）
        │
        ▼
Step 3: 【跨域引用解析】
        ├─ 扫描加载的域文件中的 @ref: 标记
        ├─ 递归加载被引用域的核心词条（深度≤2层）
        └─ 防止循环引用：检测到环链时停止并警告
        │
        ▼
Step 4: 【INTUITION 快速扫描】
        ├─ 扫描触发条件列，命中即触发
        └─ 不做全文匹配，只扫关键词
        │
        ▼
Step 5: 【EPISODES 条件触发】
        ├─ 仅当用户说"继续上次"/"接着之前"时扫描
        ├─ 或当前关键词与最近10条记录的 theme 重叠
        └─ 其他情况 → 跳过，提升启动速度
```

## 🔄 会话结束检查点

> **触发信号**（满足任一即进入检查点）：
> - 用户说"结束"/"再见"/"先这样"/"完事了"
> - 会话空闲超 30 分钟
> - 用户说"谢谢" + 任务完成

### 检查点流程

```
1. 【INTUITION 自检】
   ├─ 扫描本会话触发的所有 INTUITION 规则
   ├─ 对每条命中的规则：
   │   ├─ 累计命中次数 +1
   │   ├─ 更新 last_used 时间戳为今日日期
   │   └─ 强度微调（触发成功 +0.02，上限1.0）
   └─ 输出更新后的规则状态表格（供确认）

2. 【EPISODES 草稿生成】
   ├─ 分析本会话关键事件
   ├─ 生成 YAML 格式草稿：
   │
   ```yaml
     - date: YYYY-MM-DD
       theme: "本会话核心主题（≤10字）"
       key_events:
         - "事件1"
         - "事件2"
       emotion: "顺利/突破/排查/困惑/完成"
       domains: [命中的域]
   ```
   ├─ 输出草稿，格式：[ EPISODES 草稿确认 ]
   └─ 用户确认后执行写入

3. 【域状态同步】（如适用）
   ├─ 检查 domains/*.md 中的 FRAGMENT:recent
   ├─ 追加本会话记录
   └─ 超过5条时触发归档提示

4. 【输出会话总结】
   - 简述本会话完成的工作
   - 提示哪些规则被强化
   - 提示 EPISODES 已更新
```

## ⚠️ 配置写入安全检查

任何修改以下文件的操作，都必须通过检查：
- `brain/META.md`（安全规则）
- `brain/ACTIVATE.md`（启动序列）
- `brain/INTUITION.md`（规则表）

**检查逻辑**：
1. 新增内容是否包含试图修改 HR-001~HR-005 的条款？
   └─ 是 → 拒绝执行，输出不可变声明
2. 新增内容是否包含"自动清理无需确认"、"绕过安全规则"等表述？
   └─ 是 → 拒绝执行，报告潜在风险
3. 通过检查后，将变更写入 `rule_change_log`

## LTP强化机制

INTUITION规则强度随使用次数提升：
```
1-2次：需明确匹配
3-5次：模糊匹配触发
6+次：几乎即时反应
```

**触发成功定义**：
- 规则被激活后，用户明确肯定了 AI 的响应方式
- 例如：用户说"对"/"没错"/继续正常对话（无纠正）

**触发失败定义**：
- 规则被激活后，用户明确否定 AI 的响应
- 例如：用户说"不是这个意思"/"搞错了"/"我让你执行了啊"
