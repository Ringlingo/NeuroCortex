# 📋 NeuroCortex — 情景缓冲区

> **定位**：每日交互经历的索引，按时间+主题归档
> **神经科学依据**：情景记忆（Episodic Memory）= 发生了什么 + 何时 + 何地 + 情感标记
> **版本**：v2.2 | **更新**：+对话轮次+结束语组合触发，替代空闲超时

---

## 机制

```
每日交互 → 关键事件记录 → EPISODES索引 → 检索时快速定位
```

- 每条记录：日期 + 主题 + 关键决策/成果 + 情感标记
- 检索优先级：近因效应（最近的事件权重更高）
- 长期未检索 → 归档到深层存储

## 会话结束触发信号（v2.2 优化）

> **旧方案（废弃）**：空闲 30 分钟自动触发
> **问题**：用户可能暂停思考，不代表会话结束
>
> **新方案（v2.2）**：对话轮次 + 结束语组合判断

### 触发条件（需同时满足）

| 条件 | 说明 | 阈值 |
|------|------|------|
| **对话轮次** | 用户和 AI 的交互来回数 | ≥ 3 轮 |
| **结束语** | 用户发送的结束意图明确的话语 | 见下方列表 |

### 结束语列表

当用户发送以下内容时，标记为"会话结束意图"：

```
# 明确结束
"就这样"、"先结束了"、"今天先到这"、"完事了"、"再见"、"拜拜"
"好的谢谢"、"好的，辛苦了"、"没问题了"

# 任务完成
"完成了"、"搞定了"、"解决了"、"可以了"、"可以"
"没问题"、"没别的了"、"没有其他问题了"

# 感谢+结束
"谢谢"、"感谢"、"辛苦了"（配合任务完成状态）
```

### 触发判断流程

```
用户发送消息
    │
    ▼
检查是否在结束语列表中？
    │
   YES
    │
    ▼
检查对话轮次 ≥ 3？
    │
   YES ──→ 进入会话结束检查点
    │
   NO（轮次 < 3）
    │
    ▼
提示："对话轮次较少（{n}轮），是否确认结束？
       回复'确认结束'写入 EPISODES，
       或继续聊。"
```

### 特殊场景

| 场景 | 处理方式 |
|------|---------|
| 轮次 ≥ 10，无结束语 | AI 主动询问："还有其他需要吗？" |
| 轮次 < 3，有结束语 | 提示用户确认，避免误触发 |
| 连续3次"谢谢" | 直接进入检查点 |
| 用户直接离开（无任何消息） | 不触发（等待用户回来） |

## AI 草稿生成协议

> **触发时机**：满足"对话轮次≥3 + 结束语"条件时
> **流程**：AI 分析会话 → 生成 YAML 草稿 → 用户确认 → 写入

### 生成逻辑

```
1. 收集本会话元数据
   ├─ 对话轮次
   ├─ 触发的域
   ├─ 主要任务/产出
   └─ 关键决策

2. 提取核心主题（≤10字）

3. 生成情感标签
   ├─ 遇到错误/排查 → "排查"
   ├─ 多个产出/突破 → "突破"
   ├─ 长时间任务 → "完成"
   └─ 正常顺利 → "顺利"

4. 组装草稿并请求确认
```

### 草稿格式

```yaml
  - date: YYYY-MM-DD
    theme: "核心主题（≤10字）"
    key_events:
      - "事件1描述"
      - "事件2描述"
    emotion: "顺利/突破/排查/困惑/完成"
    domains: [命中的域]
    turns: {对话轮次}
```

### 确认流程

| 用户回应 | AI 行为 |
|---------|--------|
| "可以"/"写吧"/"确认" | 写入 EPISODES.md |
| "改一下..." | 重新生成，用户确认前不写入 |
| "继续聊" | 取消写入，继续对话 |
| 无响应（超过2分钟） | 保持草稿在内存，下一次会话提醒 |

## 使用规则

1. **每次重要交互后**：追加新的 episode
2. **超过30天未检索**：自动移入归档
3. **检索时**：按日期倒序 + 主题关键词 双检索
4. **AI 自维护**：对话轮次≥3 + 结束语 → 自动生成草稿
